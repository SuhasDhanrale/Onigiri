## 1. Executive Summary

### Current State

The `HubTestScreen.jsx` implements a basic branching map with 6 node types but lacks:

- Meaningful reward differentiation between nodes
- A coherent economy linking combat to meta-progression
- Strategic trade-offs in path selection
- Long-term engagement hooks

### Design Direction (v1.2 Updates)

**Key Conceptual Changes:**
- **Command Economy Refinement:**
  - **Base Command:** Per-node allocation (starts ~100, grows through events/upgrades)
  - **Combat Command:** Generated by killing enemies during combat (temporary)
  - **Leftover Command:** Lost after each node (no hoarding)
  - **Shop:** Uses Base Command to buy temporary buffs (1-2 combat duration)
- **No Hard Cap:** Base Command soft-limited by economic design (limited nodes, event RNG)
- **No Healing Focus:** The game does not emphasize healing mechanics. Units are expendable; survival is through tactical play.
- **Elite Wave Abilities:** Enemy waves have special abilities (poison ponds, red thunder, exploding bombers).
- **Necromancer Power:** Onmyoji Shrine offers necromancy - revive dead troops (yours or enemies) with reduced HP.

### Design Goals

| Goal | Metric |
|------|--------|
| **Agency** | Every node choice feels meaningful (not random) |
| **Momentum** | Pacing tension between safe vs risky paths |
| **Verticality** | Short-term combat rewards vs long-term meta unlocks |
| **Replayability** | Each run feels different through variation |

### Key Innovation

Your game has a unique advantage: **tower defense combat** sits under the **meta layer**. This means rewards can affect:

- **Combat** (temporary buffs/debuffs)
- **Economy** (command/honor multipliers)
- **Meta** (permanent unlocks)

## 3. Node Type Taxonomy

### 3.1 Overview

| Node Type | Icon | Threat Level | Primary Reward | Player Tension |
|-----------|------|--------------|----------------|----------------|
| `combat` | ⚔️ | Low-Med (1-3) | Command + Honor | Resource management |
| `elite` | 💀 | High (4-6) | Rare items | Risk vs survival |
| `event` | ❓ | None | Choice-based | Strategic trade-off |
| `shop` | 💰 | None | Spend Command | Opportunity cost |
| `rest` | ⛺ | None | Buff + Blessing | Pacing / urgency |
| `boss` | 👹 | Max (7) | Campaign reward | Summit moment |

> **Note:** Rest nodes provide buffs and blessings, not healing (healing is deprioritized in this design).

```

├── Event:       2 nodes (17%)
```

### 3.3 Combat Nodes — "The Crucible"

#### Purpose

Primary income source, skill expression, core gameplay loop

#### Base Rewards

```javascript
COMBAT_BASE_REWARDS = {
  threat_1: { command: [15, 25], honor: [1, 2], dropChance: 0.05 },
  threat_2: { command: [25, 40], honor: [2, 4], dropChance: 0.10 },
  threat_3: { command: [40, 60], honor: [3, 5], dropChance: 0.15 },
}

// NOTE: Command earned in combat is "field command" - temporary during battle
// Only base allocation and progression-earned command persists between nodes
```

#### Combat Variants

| Variant ID | Name | Threat | Command | Honor | Special |
|------------|------|--------|---------|-------|---------|
| `bandit_camp` | Bandit Camp | 1 | 20 | 2 | Standard |
| `ambush` | Forest Ambush | 1 | 25 | 1 | Units start closer |
| `patrol` | Patrol Route | 2 | 30 | 2 | Timed: bonus for fast clear |
| `garrison` | Forward Garrison | 2 | 15 | 5 | Enemies behind barricades |
| `caravan` | Merchant Caravan | 1 | 40 | 1 | Extra command drops |

#### Combat Node UX

- Hover shows: Enemy count, Wave count, Estimated threat
- Completion shows: Command earned, Honor earned, Drops obtained
- Quick-select: Click to preview, double-click to enter

### 3.4 Elite Nodes — "The Trial"

#### Purpose

High-risk encounters that test player skill and offer rare rewards

#### Elite Design Rules

1. **Threat always 4-6** — Never trivial
2. **Single powerful enemy OR coordinated squad** — Not a horde
3. **Unique mechanic** — Something not seen in normal combat
4. **Guaranteed rare reward** — No RNG on the big prize

#### Elite Variants

| ID | Name | Threat | Unique Mechanic | Guaranteed Reward |
|----|------|--------|-----------------|-------------------|
| `tengu_master` | Tengu Master | 4 | Flies, dodges 50% arrows | +25 Honor |
| `oni_warlord` | Oni Warlord | 5 | 3x HP, 0.5x speed, rage mode | +40 Honor |
| `shinobi_squad` | Shinobi Squad | 4 | 5 assassins, target buildings | Squad Cap +1 |
| `onmyoji_ritual` | Onmyoji Ritual | 5 | Summons reinforcements | +30 Honor |
| `yamabushi` | Yamabushi Monk | 4 | Heals self, high damage | Blessing Choice |
| `ronin_duel` | Masterless Ronin | 4 | 1v1 duel challenge | +30 Honor |

#### Elite Reward Structure

```
ELITE_REWARDS = {
  GUARANTEED: one of → +25-40 Honor | Squad Cap +1 | Unique Blessing
  BONUS:      command: [50, 80], honor: [15, 25]
  CHANCE:     25% for additional random blessing
}
```

#### Elite Wave Special Abilities (Planned)

Elite waves feature unique environmental and enemy abilities:

| Ability | Description | Elite Type |
|---------|-------------|------------|
| `poison_ponds` | Toxic pools that damage units over time | Swamp-based elites |
| `red_thunder` | Enemy version of player's thunder spell | Onmyoji elites |
| `exploding_bombers` | Enemies that detonate on death, dealing AOE | Saboteur squads |
| `necromancer_zombie` | Fallen units revive as zombies (see Necromancer) | Onmyoji Shrine |

> **Future Feature:** Full elite wave ability system pending implementation.

### 3.5 Event Nodes — "Crossroads of Fate"

#### Purpose

Break combat monotony, create stories, force meaningful trade-offs

#### Event Design Pattern

Every event presents **2-3 choices** with clear trade-offs:

- **Safe option** — Low risk, low reward
- **Risky option** — High risk, high reward
- **Moral option** — Honor-focused, may cost command

#### Event Categories

| Category | Theme | Tension Type |
|----------|-------|--------------|
| **Combat-adjacent** | Ronin, Deserters | Risk vs safety |
| **Economic** | Merchants, Villages | Command vs Honor |
| **Mystical** | Shrines, Spirits | Blessing vs Curse |
| **Moral** | Prisoners, Refugees | Honor vs Power |
| **Gambit** | Caches, Deals | Known vs Unknown |

### 3.7 Rest Nodes — "War Camp"

#### Purpose

Pacing checkpoint, recruit setup for next combat

#### Rest Options (Recruit Cap)

```
REST_OPTIONS = {
  RECRUIT_CAP: {
    name: 'Establish Garrison',
    effect: 'set_base_spawn',
    desc: 'Set units that auto-spawn at start of next combat'
  },

  REMOVE_CURSE: {
    name: 'Purification Rites',
    effect: 'remove_1_curse',
    desc: 'Remove 1 curse of your choice'
  },

  BLESSING: {
    name: 'War Council',
    effect: 'choose_blessing_from_2',
    desc: 'Choose 1 blessing from 2 options'
  }
}
```

#### Recruit Cap System

When player chooses "Establish Garrison", they select units to auto-spawn at the START of the next combat:

| Garrison Size | Cost | Effect |
|---------------|------|--------|
| **Small Garrison** | Free | 2 Ashigaru auto-spawn next combat |
| **Medium Garrison** | 30 Command | 2 Ashigaru + 1 Yumi Archer auto-spawn |
| **Large Garrison** | 60 Command | 2 Ashigaru + 1 Yumi Archer + 1 Cavalry auto-spawn |

**Rules:**
- Auto-spawned units appear immediately when combat starts (before first wave)
- Does NOT count against your squad cap
- Only applies to the NEXT combat (one-time use)
- Player can only set one garrison per Rest node

> **Note:** Healing removed as an option (healing is deprioritized). Garrison provides guaranteed early defense instead.

#### Rest Design Rules

1. **Player picks 1 option** — Cannot take all
2. **Always beneficial** — No risk
3. **Limited per campaign** — Max 1-2 rest nodes
4. **Strategic placement** — Before boss or after difficult elite gauntlet

#### Blessing Pool (Rest Node Exclusive)

## 4. Economy Architecture

### 4.1 Currency Model

```
┌─────────────────────────────────────────────────────────────────────┐
│                         CURRENCY SYSTEM                             │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  COMMAND (Authority / Chi / Mana)                                  │
│  ════════════════════════════════════                               │
│                                                                     │
│  TWO FORMS OF COMMAND:                                              │
│                                                                     │
│  ┌─────────────────────────────────────────────────────────────┐   │
│  │ BASE COMMAND (Per-Node Allocation)                          │   │
│  ├── Starting value: ~100 (adjusted by provisions)             │   │
│  ├── Increases from: Events, node choices, meta upgrades       │   │
│  ├── No hard cap: Soft-limited by economic design              │   │
│  ├── Used in: Shop purchases, event payments                    │   │
│  └── Resets each node (leftover lost)                          │   │
│  └─────────────────────────────────────────────────────────────┘   │
│                                                                     │
│  ┌─────────────────────────────────────────────────────────────┐   │
│  │ COMBAT COMMAND (Generated During Fight)                     │   │
│  ├── Generated by: Killing enemies (existing system)           │   │
│  ├── Spent on: Units, spells during combat                      │   │
│  ├── Temporary: Only exists during that combat                  │   │
│  └── Lost after: Combat ends (does not persist)                │   │
│  └─────────────────────────────────────────────────────────────┘   │
│                                                                     │
│  KEY RULE: Leftover Command is LOST after each node               │
│                                                                     │
│  ─────────────────────────────────────────────────────────────────  │
│                                                                     │
│  HONOR (Long-Term / Meta / Persistent)                             │
│  ════════════════════════════════════                               │
│  Sources:                                                           │
│  ├── Combat victories (small amounts)                              │
│  ├── Elite victories (significant)                                  │
│  ├── Moral event choices                                           │
│  ├── Boss victory (large)                                          │
│  └── Meta achievements                                             │
│                                                                     │
│  Sinks:                                                             │
│  ├── Provision unlocks (items, starting bonuses, tower upgrades)   │
│  ├── Cosmetic unlocks                                              │
│  └── Difficulty mode purchases                                      │
│                                                                     │
│  Character:                                                         │
│  ├── PERSISTS across runs                                          │
│  ├── Unlocks permanent power                                        │
│  └── Creates long-term progression                                  │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

### 4.2 Command Flow Per Node

```
COMMAND_FLOW_PER_NODE:
│
├── COMBAT NODE
│   ├── Enter with: Base Command (e.g., 100)
│   ├── Kill enemies → Generate Combat Command (+50)
│   ├── Total available: Base + Combat (150)
│   ├── Spend on units/spells (-90)
│   ├── Survive → Earn Honor (NOT Command)
│   └── End of node: Leftover Command LOST, reset to Base
│
├── SHOP NODE
│   ├── Enter with: Base Command (e.g., 100)
│   ├── Buy temporary buffs (costs Base Command)
│   ├── Buffs last: 1-2 combats
│   └── End of node: Leftover Command LOST
│
├── ELITE NODE
│   ├── Same as Combat, but harder
│   ├── Better Honor rewards
│   └── May increase Base Command on victory
│
├── REST NODE
│   ├── Option: Recover/increase Base Command capacity
│   └── Prepare for next challenge
│
└── EVENT NODE
    ├── May reward: +Base Command bonus
    └── May cost: Command for choices
```
